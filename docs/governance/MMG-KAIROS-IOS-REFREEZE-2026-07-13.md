# MMG/Kairos IOS Re-Freeze — 2026-07-13

## Status
The MMG/Kairos IOS operating architecture is re-frozen as the governing implementation baseline.

## Locked experience doctrine
- Preserve the current Command Center shell: Kairos header, analytics, parent cards, child cards, and established visual hierarchy.
- Refine the interface through simpler language, clearer business outcomes, and one-action workflows rather than structural redesign.
- Parent cards represent business areas. Each parent exposes one to five executable child actions and never more than five.
- Hide technical implementation detail behind Details, Evidence, Advanced, or History surfaces.
- The Admin Command Center and Customer Portal use one shared experience system with permission-based capability differences.
- Customer-facing terminology is the default even in the admin experience.

## Locked operating cycle
1. The user states an objective.
2. Kairos researches, prepares, produces, verifies, and stages the deliverable.
3. Decision-ready work enters the morning or evening approval brief.
4. The user chooses Approve, Deny, Fix, or View Evidence.
5. Approved work executes through the appropriate governed adapter.
6. Kairos verifies the authoritative result, stores receipts and decision history, updates the appropriate registry or Knowledge Library record, and continues dependent work.
7. Denied work closes or pauses with a decision record. Fix requests return to the responsible workflow and re-enter approval after correction.

## Authority model
- Automatic: monitoring, diagnostics, evidence collection, safe high-confidence repairs, staging preparation, and internal production.
- Governed: customer-facing copy, lifecycle routing, product presentation, visual exceptions, and deployment-ready packages.
- Executive approval: live publication, structural redesign, pricing, legal claims, destructive actions, and uncertain material changes.

## Immediate build sequence
1. Verify and stabilize Website Retool deployment and execution.
2. Build the autonomous Website Intelligence Supervisor.
3. Complete product-asset production engines.
4. Complete connector-independent social production engines.
5. Refine the Command Center into the shared customer-facing execution shell.
6. Add TikTok and other external publishing connectors in the connector phase.

## Change control
This baseline remains frozen. Material architectural changes require explicit executive approval and a recorded amendment. Implementation may continue aggressively inside these boundaries.
