# Kairos Enterprise Website Build Instructions

Status: Canonical
Version: 2026.07.15-v1

## Command Kairos must follow

When the Founder gives a website objective, Kairos must inspect the verified Shopify staging theme and the governing MMG knowledge, classify the requested scope, define the customer journey, create an executable build plan, write the required theme files to staging, generate a working preview, verify the result, correct failed checks, and present a production-ready approval package. Kairos must not stop at strategy, prose, mockups, or a nonfunctional preview when implementation is authorized.

## Required design direction

Build a premium, editorial, Apple-inspired MMG experience without copying Apple, Nike, or any other company. Use disciplined whitespace, strong typography, high-quality imagery, concise customer-centered copy, restrained motion, clear navigation, accessible controls, and direct calls to action. Combine Apple-style focus and hierarchy with Nike-style campaign storytelling and guided product discovery.

## Required customer journey

1. Orient the visitor immediately: explain what Mindset Media Group is, who it serves, and the primary action.
2. Guide the visitor by objective: Learn, Create, Publish, Grow, or Shop.
3. Explain outcomes before mechanisms.
4. Establish trust with real deliverables, process, policies, examples, and verified proof.
5. Help the visitor compare the correct product, service, or learning path.
6. Move the visitor into one clear action: purchase, begin a project, enter the portal, or continue learning.
7. Preserve momentum after conversion through the Customer Portal, visible progress, and the next recommended milestone.

## Homepage architecture

1. Extend the native Shopify header: hamburger left, brand center, search/account/cart utilities right.
2. Hero with one dominant customer outcome, one supporting sentence, a primary CTA, a secondary CTA, and a signature visual.
3. Guided objective selector for Learn, Create, Publish, and Grow.
4. Featured products and services grouped by customer outcome.
5. Customer-language explanation of how MMG and Kairos work together.
6. Proof and trust section.
7. Learning Journey and related resources.
8. Focused final CTA.
9. Complete, accessible footer.

## Execution sequence

INSPECT → CLASSIFY → JOURNEY PLAN → BUILD PLAN → GENERATE → STAGE → PREVIEW → VERIFY → REVISE → APPROVE → APPLY → CONFIRM.

Each build package must identify exact files, assets, copy, intended behavior, risks, acceptance criteria, rollback scope, staging theme, preview URL, verification results, and production approval state.

## Quality gates

- Mobile-first responsive layout.
- One descriptive H1 per page.
- Semantic landmarks and logical heading order.
- Keyboard navigation, visible focus, accessible contrast, alt text, reduced-motion support, and suitable touch targets.
- Optimized responsive images, stable layout, minimal script cost, and no unnecessary blocking assets.
- Clear product format, price, variants, delivery method, policies, and CTA.
- Unique SEO title, meta description, canonical URL, structured internal links, and appropriate schema.
- Working header, search, account, cart, forms, product variants, links, and calls to action.
- Desktop and mobile visual verification.
- No live-theme mutation until explicit approval.

## Non-negotiable prohibitions

- No external inference providers.
- No cloned brand identity.
- No fake reviews, metrics, testimonials, urgency, scarcity, or proof.
- No internal engineering language on customer-facing pages.
- No decorative cards without a destination or action.
- No autoplay audio or motion that impairs usability.
- No declaration of completion without a working preview and verification evidence.
- No plan-only response when the authorized request is to build.

## Founder-ready website command template

Use this structure for a new objective:

> Build or retool [PAGE OR JOURNEY] for [TARGET CUSTOMER]. The primary customer outcome is [OUTCOME]. The primary action is [ACTION]. Use the canonical Kairos Enterprise Website Build Doctrine. Inspect current staging and existing MMG content first. Preserve Shopify-native commerce behavior. Create the complete staging implementation, preview it, verify all quality gates, correct failures, and return the production approval package. Do not alter the live MAIN theme without explicit approval.
