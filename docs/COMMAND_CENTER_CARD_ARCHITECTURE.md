# MMG/Kairos Command Center Card Architecture

Status: canonical and refrozen  
Effective: 2026-07-13

## Structural Rule

The Kairos Command Center contains exactly five primary parent cards:

1. Knowledge
2. Content
3. Business
4. Customers
5. Operations

Each parent card contains exactly five child cards. The system therefore exposes exactly 25 primary work entry points.

## Containment Rule

Every user-facing tool, studio, workflow, control, library, queue, registry, or production surface must have one defined home inside one of the five parent cards.

Standalone floating launch buttons, detached action pills, persistent edge controls, and unassigned tool launchers are prohibited.

Overlays and focused workspaces may open after a child card is selected, but their entry point must remain inside the parent/child card architecture.

## Capacity Rule

The Command Center must not exceed:

```text
5 parent cards × 5 child cards = 25 entry points
```

When a new capability is introduced, it must be handled through one of these methods:

- expand an existing child tool;
- combine overlapping capabilities into a single coherent child tool;
- replace a lower-value child entry point through explicit architecture review;
- route the capability through an existing workspace, registry, or production center.

Adding a sixth parent card, a sixth child card, or a floating launcher is not permitted without explicit executive amendment.

## Experience Standard

The card architecture must remain:

- professional and visually integrated;
- mobile-first and responsive;
- clear enough that users can locate a capability without memorizing prompts;
- stable as capabilities deepen;
- free of detached controls that obscure content or compete with navigation;
- organized around objectives and completed work rather than isolated tools.
