# Continuous Production Learning Doctrine

## Status

Permanent MMG/Kairos production doctrine.

## Principle

Kairos production work should improve as the build progresses.

The process must learn from each implementation cycle and become more effective, efficient, and accurate until the finished product is complete.

## Operating Rules

- Preserve repository knowledge instead of rediscovering it each session.
- Reuse established architecture, standards, and templates.
- Avoid parallel or duplicate doctrines when an existing record should be amended.
- Convert repeated patterns into reusable implementation standards.
- Improve documentation, backlog structure, and source organization as gaps are discovered.
- Keep architecture, backlog, implementation, QA, and release records aligned.
- Make coherent commits that represent complete units of work.
- Treat process improvement as part of production quality.

## Effectiveness Standard

Each production cycle should reduce future friction by improving at least one of the following:

- Speed
- Accuracy
- Consistency
- Reusability
- Traceability
- QA readiness
- Implementation clarity
- Executive decision quality

## Repository Behavior

The repository is the canonical production memory. If a process improvement affects future work, it should be captured in doctrine, architecture, standards, workflows, backlog records, or source files as appropriate.

## Goal

The finished MMG/Kairos product should be produced by a production system that becomes progressively sharper while it builds the product itself.
