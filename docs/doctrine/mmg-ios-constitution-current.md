# MMG IOS Constitution - Current Baseline

Status: Frozen unless explicitly amended.

## Core Identity

Mindset Media Group is the parent publishing and creator-education company. Kairos is the flagship operating platform being built inside the MMG ecosystem. The current Shopify website is Version 1 of the operating system and the proving ground for future Kairos platform behavior.

## Operating Doctrine

MMG is treated as an active enterprise software and publishing company under continuous professional development. Future work defaults to implementation, refinement, validation, and production-ready vertical slices instead of abstract planning.

## Portal-First Doctrine

The MMG website, customer portal, admin portal, publishing workflows, production workflows, growth workflows, quality workflows, and release workflows are one connected operating system. External tools are auxiliary unless explicitly promoted. GitHub is the canonical source repository. Shopify remains the current live deployment surface.

## Kairos Doctrine

Kairos exists to help transform knowledge, experience, purpose, and ideas into professional assets at the right moment of opportunity. Platform language should reinforce purpose, transformation, stewardship, disciplined execution, growth, legacy, and meaningful opportunity.

## Production Mode Doctrine

When the user gives a production trigger such as MMG, resume backlog, let's go, or equivalent, execute the locked pipeline and return the finished deliverable. Avoid progress reports, process narration, readiness statements, and excuses unless a genuine blocker makes completion impossible.

## Full-Code Preservation Doctrine

When full page or product code is provided, treat it as the authoritative source candidate. Never return or store shortened, representative, partial, placeholder, excerpted, or compressed production code as the canonical version. Shipped source must be complete and copy-and-paste ready.

## Clean URL Doctrine

Public MMG URLs should remain clean and canonical by default. Customer-facing links, navigation items, buttons, emails, product pages, and portal links should avoid unnecessary anchors, hashes, query clutter, or malformed routing.

## Canonical Navigation

Main navigation:

- Home
- Catalog
- Publishing Services
- Knowledge Library
- Customer Portal
- Company
- Contact

Company dropdown:

- About Mindset Media Group
- Our Standards
- Publishing Philosophy
- Founder
- The MMG Promise

Customer Service is not part of the main navigation unless explicitly reintroduced.

## Page Architecture Doctrine

Approved pages use the left-aligned editorial page architecture unless a specific template requires another layout. Pages should preserve the premium MMG visual language, spacing, typography, clean Apple-style rhythm, blue accent system, and ecosystem continuity.

## Homepage Golden Master

The approved homepage sequence is:

Hero -> Guided Path -> Build Options -> Learning Resources -> Free Toolkit -> Connected Ecosystem -> Production Pathway -> Road to 1M -> Social -> Founder -> Choose Next Step.

The homepage is experience-first, journey-first, and ecosystem-first.

## Product Page Doctrine

Standard single-variant product pages use the approved MMG-AI-PROMPTING-FOR-BEGINNERS-v1.3.1 carousel service price fix base unless explicitly superseded.

Multi-variant products use the approved Publish-Ready Book Build Service v1.9 editorial premium UX framework. Variant cards must display the package name, price, scope descriptor, details, variant-specific action, and proper Shopify variant behavior.

## Carousel Doctrine

The canonical carousel is MMG Intelligent Carousel Engine v2.0: autonomous one-card advancement, no visible manual arrows by default, smooth eased transition, swipe or drag support, automatic resume after interaction, infinite loop, Shopify catalog-driven product loading, current-product exclusion, intelligent recommendation ordering, and full image containment.

## Admin Portal Doctrine

The approved Admin Portal is the canonical administrative workspace. It governs future MMG administrative interfaces and should preserve the same visual language, spacing, typography, portal rhythm, and premium command-center language.

## Professional Publishing Operating System Doctrine

Kairos must function as a professional publishing operating system, not a simple customer portal or file upload tool. It should support the operational needs of a modern publishing company, digital media company, creator-education company, and service production engine.

## Foundational Vertical Slices

Foundational command centers include:

- Admin Operations Center
- Customer Portal
- Professional Publishing Command Center
- Production Command Center
- Quality and Release Center
- Automation and Intelligence Center
- Growth, Marketing and Advertising Command Center

## Growth, Marketing and Advertising Doctrine

Marketing, advertising, promotions, customer acquisition, retention, and campaign management are first-class operational capabilities. Human approval is required before external campaign launch unless explicitly changed.

Capabilities include campaign planning, promotion lifecycle management, discount and promo-code generation, marketing calendar, campaign registry, email orchestration, landing-page recommendations, cross-sell and upsell generation, audience segmentation, lifecycle marketing, and creative brief generation.

## Editorial Publishing Engine Doctrine

Editorial content is a first-class manufacturing workflow. The Editorial Publishing Engine generates production-ready Shopify blog deliverables including HTML source, SEO metadata, internal linking recommendations, QA documentation, release notes, packaging artifacts, and a branded blog cover image when applicable.

## Version 1 Launch Backlog Doctrine

Before revising the canonical onboarding PDF or continuing service-product backlog manufacturing, perform live main-navigation and critical-link validation on the public MMG site. Broken destinations become immediate Phase 1 priority.

## Repository Rule

This GitHub repository is the canonical long-term home for MMG IOS, Kairos, Shopify source, documentation, registries, assets, backlog, release records, and operating doctrine.
