# MMG/Kairos Homepage Preservation-First Doctrine

## Status
Approved implementation doctrine for all future homepage refinement, Website Production, autonomous website improvement, and Kairos-guided experience work.

## Core Rule
When converting the Mindset Media Group homepage into a Kairos-guided experience, preserve the existing approved visual system and structural presentation unless the executive explicitly authorizes a redesign.

Kairos must treat the current homepage as the visual baseline and modify the experience primarily through content, context, routing, backend intelligence, and verified functional improvements.

## Preserve Exactly
Unless a specific change is requested, preserve:

- overall page layout and section order
- visual hierarchy
- spacing and section rhythm
- black and MMG blue design language
- existing color relationships
- typography scale and weight relationships
- pills, badges, buttons, cards, outlines, and radii
- approved header structure and behavior
- image framing and media placement
- responsive behavior and mobile proportions
- transitions, reveals, and restrained visual effects
- existing Shopify-native maintainability
- the recognizable character and feel of the approved MMG homepage

Do not replace the page with a generic AI landing page, agency layout, white-background system, unrelated visual language, or wholesale redesign.

## Allowed Changes
Kairos may:

- replace or refine text where needed to establish the Kairos-guided experience
- clarify MMG as the overarching ecosystem and Kairos as the intelligence operating system
- improve calls to action and customer pathways
- connect sections to verified MMG pages, products, tools, and resources
- add selected visuals within the existing design system
- improve accessibility, SEO, performance, mobile behavior, and semantic markup
- repair broken links, dead ends, invalid routes, and backend defects
- add structured data, analytics hooks, journey tracking, verification, and rollback support
- improve content context without changing the approved visual identity

## Guided Experience Content Direction
The homepage should communicate:

- Your Knowledge Has Value.
- Mindset Media Group is the complete ecosystem.
- Kairos is the intelligence and orchestration layer that guides the user through it.
- Visitors bring an objective; Kairos organizes the path forward.
- MMG helps users discover, organize, build, publish, market, and grow durable assets from their knowledge and experience.

This content should be integrated into the existing visual and structural system rather than imposed through a replacement design.

## Execution Standard
Before changing the homepage, Kairos must:

1. Inspect the current approved homepage and record its structure, styles, section order, colors, spacing, components, and responsive behavior.
2. Classify each proposed change as content-only, context-only, routing, backend, functional, visual enhancement, or explicit redesign.
3. Default to content-only, context-only, routing, backend, and functional changes.
4. Reject or escalate any proposal that would materially alter the approved visual system without explicit executive authorization.
5. Apply all changes to Kairos Staging first.
6. Provide a preview, refinement option, visual verification, approval gate, publication control, and rollback evidence.

## Refinement Prompt Rule
For hybrid homepage refinement, use this governing instruction:

"Preserve the current approved Mindset Media Group homepage exactly in its visual design, structure, spacing, colors, typography relationships, pills, cards, buttons, imagery treatment, responsive behavior, section rhythm, and overall character. Do not redesign the page. Convert it into a Kairos-guided experience by updating only the content, context, customer pathways, verified links, backend behavior, accessibility, SEO, performance, and selected visuals where they fit naturally inside the existing design system. Mindset Media Group remains the ecosystem; Kairos becomes the intelligence and orchestration layer. Use the current homepage as the visual source of truth. Any material visual redesign requires separate executive approval."

## Autonomous Engine Rule
The future Autonomous Growth Engine may continuously identify homepage improvements, but it must not gradually erode the approved visual identity through repeated autonomous edits.

Every proposed website Action Card must declare whether it preserves the visual baseline. Visual drift must be detected and treated as a defect.

## Publication Boundary
No homepage change may reach the live theme without:

- staging execution
- rendered preview
- visual review
- explicit executive approval
- governed release preparation
- production verification
- retained rollback capability
