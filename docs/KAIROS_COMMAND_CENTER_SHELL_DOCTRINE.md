# Kairos Command Center Shell Doctrine

Status: Canonical and locked
Approved: 2026-07-11

## Canonical visual shell

The live recovery-mode shell established in build `kairos-runtime-reset-20260711-1` is the permanent visual foundation for the Kairos Command Center.

Preserve the following elements unless explicitly superseded by executive approval:

- The official Kairos logo banner at the top of the experience.
- A black-first cinematic background.
- Cyan-blue borders, highlights, and status accents.
- Large rounded operational cards with generous spacing.
- Oversized, highly legible mobile typography.
- A full-width mobile-first layout.
- Clear status hierarchy and visible build/runtime identification.
- Restrained glass, metal, and luminous interface treatments.
- Minimal navigation friction and no cramped desktop-first controls.

## Stability rule

The runtime may evolve beneath this shell, but the shell itself must remain visually stable. New capability modules, status views, approval flows, evidence panels, and production controls must be integrated into this design language rather than replacing it with a different interface framework.

## Header rule

The Kairos logo banner is a permanent top-level brand element of the Command Center. It must remain prominent, correctly proportioned, mobile-safe, and visually separated from operational content.

## Experience rule

The shell must communicate confidence, clarity, and operational seriousness. It should feel like one cohesive intelligence operating system rather than a collection of unrelated tools or admin panels.

## Change control

Any future redesign that materially alters the banner, black-first composition, card architecture, mobile typography, border language, or overall cinematic character requires explicit executive approval before implementation.
