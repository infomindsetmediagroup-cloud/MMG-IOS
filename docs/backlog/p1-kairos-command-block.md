# P1: Kairos Command Block / AI Work Queue Intake

## Priority
P1

## Placement
Top of Kairos dashboard.

## Purpose
Create a natural-language command block that lets the user enter operational instructions directly into Kairos while the iOS app is still under construction.

## Supported commands
- Create or add a product
- Make a website change
- Add or update an image
- Add or update a product listing
- Queue Shopify, publishing, marketing, content, dashboard, or operational work

## Required behavior
- Commands become queued actionable tasks, not static notes.
- Preserve the original command.
- Parse each command into a structured work item.
- Assign status, priority, category, dependencies, and execution notes.
- Display recent commands and queue status on the dashboard.

## Initial statuses
- New
- Analyzing
- Ready
- Queued
- In Progress
- Waiting
- Review
- Completed
- Failed
- Cancelled

## Initial categories
- Product Operations
- Website Operations
- Shopify Operations
- Content Operations
- Growth & Marketing
- Kairos Engineering
- General Operations
