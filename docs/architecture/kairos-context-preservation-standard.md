# Kairos Context Preservation Standard

## Status
Approved architecture.

## Purpose
Ensure work retains continuity across sessions, departments, approvals, and production cycles.

## Context Model
Every project, task, asset, workflow, and decision should maintain persistent context including:
- Current objective
- Background and rationale
- Related work items
- Dependencies
- Current lifecycle state
- Approval history
- Decisions made
- Outstanding questions
- Next recommended action
- Responsible department

## Operating Rules
- Preserve context instead of requiring rediscovery.
- Carry approved decisions forward automatically.
- Surface only context relevant to the current executive decision.
- Maintain complete historical context for drill-down and audit.
- Transfer context with every cross-department work package.

## Success Metric
The executive should never need to repeatedly explain the same project context to Kairos as work progresses.
