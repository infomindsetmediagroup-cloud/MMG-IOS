# Backlog

This folder stores production backlog records, implementation queues, blockers, and execution sequencing.

## Subfolders

```text
active/       Current active production items.
blocked/      Blocked items and dependency notes.
planned/      Approved future work that has not started.
completed/    Completed backlog records and shipped item summaries.
```

## Rule

Backlog records should connect to source files, QA records, and release records when applicable.
